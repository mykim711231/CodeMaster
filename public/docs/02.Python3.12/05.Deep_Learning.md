## 핵심 개념
> PyTorch의 `Tensor`는 NumPy `ndarray`와 유사하지만 GPU 가속이 가능한 다차원 배열로, `autograd` 엔진이 모든 연산의 미분을 자동 계산하여 역전파(Backpropagation)를 구현한다. `nn.Module`을 상속해 신경망 레이어와 `forward()`를 정의하고, `optimizer`(SGD, Adam 등)로 경사하강법을 수행하며, `DataLoader`로 미니배치를 반복 공급하며 training loop를 구성한다. `torch.cuda.amp`(Automatic Mixed Precision)로 FP16 연산을 혼합해 메모리 절약과 속도 향상을 도모하고, `torch.compile()`(PyTorch 2.0+)로 그래프 최적화를 적용한다. TensorFlow/Keras는 `Sequential`, `compile`, `fit`의 고수준 API로 빠른 프로토타이핑을 지원하며, `tf.data`로 대용량 데이터 파이프라인을 구축한다.

## 학습 목표
- PyTorch `Tensor`를 생성하고 GPU로 연산할 수 있다.
- `nn.Module`을 상속하여 신경망을 정의하고 `forward()`를 구현할 수 있다.
- `optimizer`, 손실 함수, training loop로 모델을 학습시킬 수 있다.
- `Dataset`/`DataLoader`로 커스텀 데이터를 로드하고 TensorFlow/Keras로 간단한 모델을 구성할 수 있다.

## 예제 코드
```python
import torch  # ← PyTorch 메인 라이브러리
import torch.nn as nn  # ← 신경망 레이어 및 손실 함수 모듈
import torch.optim as optim  # ← 최적화 알고리즘 (SGD, Adam 등)
from torch.utils.data import Dataset, DataLoader  # ← 커스텀 데이터셋·배치 로더
import numpy as np  # ← NumPy (데이터 변환용)

# --- Tensor + GPU ---
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")  # ← GPU 사용 가능하면 cuda, 아니면 cpu
x = torch.tensor([[1.0, 2.0], [3.0, 4.0]], device=device)  # ← 2×2 텐서를 GPU/CPU에 생성
y = torch.randn(2, 3, device=device)  # ← 정규분포 난수로 2×3 텐서 생성 (동일 장치)
print("Tensor device:", x.device)  # ← 텐서가 위치한 장치 확인 (cuda:0 또는 cpu)
print("Matmul result:\n", x @ torch.ones(2, 3, device=device))  # ← 행렬 곱셈 (2×2) @ (2×3) → (2×3)

# --- nn.Module ---
class SimpleNN(nn.Module):  # ← nn.Module을 상속한 신경망 클래스 정의
    def __init__(self, input_dim: int, hidden_dim: int, output_dim: int):  # ← 레이어 정의
        super().__init__()  # ← 부모 클래스 초기화 (필수)
        self.fc1 = nn.Linear(input_dim, hidden_dim)  # ← 완전연결층1: input → hidden
        self.relu = nn.ReLU()  # ← ReLU 활성화 함수 (비선형성 부여)
        self.fc2 = nn.Linear(hidden_dim, output_dim)  # ← 완전연결층2: hidden → output

    def forward(self, x: torch.Tensor) -> torch.Tensor:  # ← 순전파 정의 (직접 호출하지 않음)
        x = self.fc1(x)  # ← 첫 번째 선형 변환
        x = self.relu(x)  # ← ReLU 활성화 적용 (음수를 0으로)
        x = self.fc2(x)  # ← 두 번째 선형 변환 → 최종 출력 (logits)
        return x  # ← CrossEntropyLoss와 함께 사용할 raw logits 반환

# --- Custom Dataset ---
class DummyDataset(Dataset):  # ← Dataset 상속: 커스텀 데이터셋 정의
    def __init__(self, n_samples: int, n_features: int, n_classes: int):
        self.X = torch.randn(n_samples, n_features)  # ← 랜덤 특성 텐서 (n_samples × n_features)
        self.y = torch.randint(0, n_classes, (n_samples,))  # ← 랜덤 레이블 (0~n_classes-1)

    def __len__(self):  # ← 데이터셋 크기 반환 (DataLoader가 len() 호출)
        return len(self.X)

    def __getitem__(self, idx: int):  # ← 인덱스로 (특성, 레이블) 쌍 반환
        return self.X[idx], self.y[idx]  # ← DataLoader가 배치 구성 시 반복 호출

# --- DataLoader ---
dataset = DummyDataset(n_samples=500, n_features=10, n_classes=3)  # ← 500개 샘플 데이터셋
dataloader = DataLoader(dataset, batch_size=32, shuffle=True)  # ← 32개씩 배치, 매 epoch 셔플

# --- Training loop ---
model = SimpleNN(input_dim=10, hidden_dim=20, output_dim=3).to(device)  # ← 모델 생성 후 GPU/CPU로 이동
criterion = nn.CrossEntropyLoss()  # ← 분류 손실 함수 (softmax + NLL을 내부 처리)
optimizer = optim.Adam(model.parameters(), lr=0.001)  # ← Adam 옵티마이저 (lr=학습률)

model.train()  # ← 학습 모드 설정 (Dropout, BatchNorm 활성화)
for epoch in range(3):  # ← 3 에폭 반복
    total_loss = 0.0  # ← 에폭별 누적 손실 초기화
    for batch_X, batch_y in dataloader:  # ← DataLoader에서 미니배치 반복
        batch_X, batch_y = batch_X.to(device), batch_y.to(device)  # ← 배치를 GPU/CPU로 전송

        optimizer.zero_grad()  # ← 기울기 초기화 (매 배치마다 필수!)
        outputs = model(batch_X)  # ← 순전파: 모델에 배치 입력 → 예측값 출력
        loss = criterion(outputs, batch_y)  # ← 손실 계산: 예측 vs 실제 레이블
        loss.backward()  # ← 역전파: 기울기(그래디언트) 자동 계산
        optimizer.step()  # ← 파라미터 업데이트: 기울기 방향으로 한 스텝 이동

        total_loss += loss.item()  # ← 배치 손실 누적 (item()으로 Python float 추출)

    avg_loss = total_loss / len(dataloader)  # ← 평균 손실 = 총합 / 배치 수
    print(f"Epoch {epoch+1:2d}, Loss: {avg_loss:.4f}")  # ← 에폭별 평균 손실 출력

# --- TensorFlow/Keras (간단 예시) ---
# import tensorflow as tf  # ← TensorFlow 임포트 (주석 해제 후 사용)
# model_tf = tf.keras.Sequential([  # ← Sequential API로 레이어 쌓기
#     tf.keras.layers.Dense(20, activation="relu", input_shape=(10,)),  # ← 은닉층
#     tf.keras.layers.Dense(3, activation="softmax"),  # ← 출력층 (확률 분포)
# ])
# model_tf.compile(optimizer="adam", loss="sparse_categorical_crossentropy", metrics=["accuracy"])  # ← 컴파일
# model_tf.fit(X_np, y_np, epochs=5, batch_size=32, validation_split=0.2)  # ← 학습
```

## 주요 패턴
- `nn.Module` + `forward()`: 레이어를 `__init__`에 정의하고 `forward()`에서 연결 순서를 기술한다.
- `optimizer.zero_grad()` → `loss.backward()` → `optimizer.step()`: 매 배치마다 이 3단계를 반복한다.
- `model.train()` vs `model.eval()`: Dropout, BatchNorm 등의 동작을 학습/추론 모드로 전환한다.
- `Dataset` + `DataLoader`: `__getitem__`으로 샘플을 반환하고 `DataLoader`가 배치, 셔플, 병렬 로딩을 처리한다.
- `to(device)`: 모델과 텐서를 동일한 장치로 전송하여 GPU 가속을 활용한다.

## 주의사항
- `forward()` 호출은 `model(x)`로 직접 하며, `model.forward(x)`는 호출하지 않는다 (Hook이 우회됨).
- 학습 전 `optimizer.zero_grad()`를 생략하면 기울기가 누적되어 학습이 불안정해진다.
- 추론 시 `torch.no_grad()` 컨텍스트 안에서 수행해야 메모리 사용량이 줄어든다.
- TensorFlow와 PyTorch의 기본 데이터 차원 순서가 다를 수 있다 (NHWC vs NCHW).
- CUDA 사용 시 텐서(`.to(device)`)와 모델(`model.to(device)`)을 별도로 전송해야 한다.
- `CrossEntropyLoss`는 내부적으로 softmax를 적용하므로, 모델 출력에 softmax를 따로 적용하면 이중 적용되어 학습이 망가진다.
- `DataLoader`의 `num_workers`를 0보다 크게 설정하면 데이터 로딩이 멀티프로세싱으로 병렬화되지만, Windows에서는 `if __name__ == "__main__"` 가드가 반드시 필요하다.
