# LLM

## Official Documentation
- [Transformers](https://huggingface.co/docs/transformers/index)

## 핵심 개념
> Hugging Face Transformers는 사전 학습된 최신 NLP/AI 모델을 `from_pretrained()` 한 줄로 로드하고 사용할 수 있게 해주는 라이브러리다. `AutoModel`, `AutoTokenizer`, `AutoConfig`로 모델 아키텍처와 토크나이저를 체크포인트 이름만으로 자동 매핑하며, `pipeline()`으로 감정 분석·텍스트 생성·NER·요약 등 태스크를 단 한 줄로 수행한다. 토큰화(Tokenization)는 텍스트를 모델이 이해할 수 있는 정수 ID 시퀀스로 변환하는 핵심 전처리 과정이며, `padding`·`truncation`·`attention_mask`로 배치 처리를 지원한다. 임베딩(Embedding)은 토큰/문장을 고정 차원의 밀집 벡터로 변환하여 의미적 유사도를 측정 가능하게 하며, `SentenceTransformer`는 문장 단위 임베딩에 특화된 경량 모델을 제공한다.

## 학습 목표
- AutoModel, AutoTokenizer, AutoConfig로 사전 학습 모델 자동 로드
- pipeline()을 이용한 감정 분석, 텍스트 생성, NER 등 태스크 수행
- 입력 텍스트의 토큰화, 패딩, 어텐션 마스크 처리 이해
- SentenceTransformer로 문장/문서 임베딩 벡터 생성
- GPU/CPU 장치 간 모델 이동 및 추론 최적화

## 예제 코드
```python
from transformers import AutoTokenizer, AutoModel, pipeline  # ← Hugging Face 핵심 클래스 불러오기

# pipeline으로 감정 분석 (단 한 줄로 모델 로드 + 추론)
classifier = pipeline("sentiment-analysis")  # ← 기본 sentiment 모델 자동 다운로드 및 로드
result = classifier("I love using Hugging Face Transformers!")  # ← 텍스트 감정 분석 실행
print(result)  # ← [{'label': 'POSITIVE', 'score': 0.999...}] 형태로 출력

# AutoTokenizer + AutoModel로 임베딩 추출
checkpoint = "bert-base-uncased"  # ← 사용할 사전 학습 모델 이름 (HuggingFace Hub에서 다운로드)
tokenizer = AutoTokenizer.from_pretrained(checkpoint)  # ← 해당 모델의 토크나이저 자동 로드
bert_model = AutoModel.from_pretrained(checkpoint)  # ← 사전 학습된 BERT 모델 가중치 로드

texts = ["Hello, world!", "Transformers are powerful."]  # ← 임베딩할 텍스트 리스트 (배치 처리)
inputs = tokenizer(  # ← 텍스트를 토큰 ID로 변환
    texts,  # ← 입력 텍스트 리스트
    padding=True,  # ← 짧은 문장에 [PAD] 토큰 추가하여 길이 통일
    truncation=True,  # ← max_length 초과 시 잘라내기
    return_tensors="pt"  # ← PyTorch 텐서로 반환 ("tf" = TensorFlow)
)
outputs = bert_model(**inputs)  # ← 모델 순전파: 토큰 ID -> 은닉 상태 벡터
# [CLS] 토큰(문장 시작 특수 토큰)의 임베딩을 문장 대표 벡터로 사용
embeddings = outputs.last_hidden_state[:, 0, :]  # ← 모든 문장[:], 첫 토큰[0], 전체 차원[:]
print(embeddings.shape)  # ← (2, 768) -> 2문장 × 768차원 임베딩

# SentenceTransformer로 문장 임베딩 (경량·고속)
from sentence_transformers import SentenceTransformer  # ← 별도 라이브러리 (pip install sentence-transformers)

st_model = SentenceTransformer("all-MiniLM-L6-v2")  # ← 경량 문장 임베딩 모델 (384차원, 빠름)
sentences = ["This is an example sentence", "Each sentence is converted"]  # ← 임베딩할 문장들
sentence_embeddings = st_model.encode(sentences)  # ← 문장 -> 벡터 변환 (정규화된 임베딩)
print(sentence_embeddings.shape)  # ← (2, 384) -> 2문장 × 384차원
```

## 주요 패턴
- Model/Task별 pipeline: `pipeline("text-generation")`, `pipeline("ner")`, `pipeline("summarization")` 등으로 즉시 추론
- 토크나이저 공통 인터페이스: `encode()`, `decode()`, `batch_encode_plus()`로 배치 처리
- Auto Classes: `AutoModel`, `AutoTokenizer`, `AutoConfig`로 체크포인트 이름만으로 자동 로드
- from_pretrained: 로컬 또는 Hugging Face Hub에서 사전 학습 가중치 다운로드
- 임베딩 추출: `outputs.last_hidden_state`에서 [CLS] 토큰 또는 평균 풀링으로 벡터 생성
- model.eval() + torch.no_grad(): 추론 시 그래디언트 비활성화로 메모리 최적화
- 저장/로드: `model.save_pretrained()`, `tokenizer.save_pretrained()`로 로컬 저장

## 주의사항
- 모델 다운로드 크기가 수 GB에 달할 수 있어 디스크 공간 확인 필요
- 토크나이저와 모델의 최대 입력 길이(`max_length`)를 초과하면 `truncation=True` 필요
- pipeline은 편리하지만 세밀한 제어가 필요한 경우 AutoModel을 직접 사용
- `return_tensors="pt"`는 PyTorch, `"tf"`는 TensorFlow 텐서 반환
- SentenceTransformer과 Transformers의 임베딩 품질/속도가 다르므로 용도에 맞게 선택
- 모델 재현성 확보 시 `seed` 고정 및 `torch.backends.cudnn.deterministic = True` 설정
- `AutoModel.from_pretrained()`는 기본적으로 모델을 CPU에 로드하므로, GPU 사용 시 `model.to("cuda")`를 호출한다.
- `tokenizer()` 호출 시 `max_length`를 지정하지 않으면 모델의 기본 최대 길이까지 허용되어 메모리 초과가 발생할 수 있다.
