# Prompt & Structured Output
## Official Documentation
- [Prompt Engineering Guide](https://www.promptingguide.ai/)
- [OpenAI Function Calling](https://platform.openai.com/docs/guides/function-calling)
- [Pydantic](https://docs.pydantic.dev/latest/)

## 핵심 개념
> 프롬프트 엔지니어링은 LLM의 출력 품질을 높이기 위해 입력을 설계하는 기법이다. Few-shot은 2~4개의 예시(Q&A 쌍)를 프롬프트에 포함하여 모델이 출력 형식과 패턴을 학습하게 한다. Chain-of-Thought(CoT)는 "단계별로 생각해보자"와 같은 유도문으로 중간 추론 과정을 명시적으로 드러내게 하여, 복잡한 추론 작업의 정확도를 크게 향상시킨다. Function Calling(Tool Calling)은 모델이 정해진 JSON Schema에 따라 호출할 함수명과 인자를 선택해 반환하게 하여, LLM과 외부 API/DB를 안전하게 연동하는 핵심 인터페이스다. Pydantic과 결합하면 함수 파라미터의 타입 검증·직렬화를 자동화할 수 있다. `response_format={"type": "json_object"}`는 모델이 JSON으로만 응답하게 강제하지만, 복잡한 스키마일수록 모델이 스키마를 무시할 확률이 높아진다. instructor 라이브러리는 이러한 과정을 `response_model` 파라미터 하나로 추상화하여 Pydantic 모델 기반 구조화 출력을 간결하게 구현한다.

## 학습 목표
- Few-shot 프롬프트로 출력 형식 제어하기
- Chain-of-Thought 프롬프트로 복잡한 추론 작업 정확도 향상시키기
- Function Calling을 통해 LLM과 외부 API 연동 구현하기
- JSON Schema로 출력 구조를 정의하고 응답 검증하기
- Pydantic 모델로 구조화된 LLM 응답 파싱 및 엄격한 타입 검증하기

## 예제 코드
```python
# Few-shot + Chain-of-Thought
from openai import OpenAI

client = OpenAI()

# ← Few-shot 예시: 1개의 Q&A 예시로 출력 패턴을 학습시켜요
prompt = """Q: 철수가 사과 5개를 가지고 있었고, 영희가 3개를 더 줬다. 몇 개인가?
단계별로 생각해보자:
1. 철수가 처음 가진 사과: 5개
2. 영희가 준 사과: 3개
3. 더하면: 5 + 3 = 8개
답: 8

Q: 가방에 책 12권이 있었는데 4권을 꺼냈다. 몇 권인가?
"""  # ← 모델이 위 예시를 참고해 동일한 "단계별->답" 형식으로 출력

response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": prompt}],
    temperature=0.0,  # ← 0.0: 항상 같은 결과를 위해 창의성 억제 (float 타입)
)
print(response.choices[0].message.content)
```
```python
# Function Calling
# ← tools: 모델이 호출할 수 있는 함수의 JSON Schema 목록을 정의
tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "현재 날씨 조회",
        "parameters": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "도시명"},
                "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
            },
            "required": ["city"]  # ← city 파라미터는 필수
        }
    }
}]

response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "서울 날씨 어때?"}],
    tools=tools,
    tool_choice="auto",  # ← 모델이 알아서 함수 호출 여부를 결정
)
# ← tool_calls[0]: 모델이 호출하기로 선택한 함수 정보 추출
tool_call = response.choices[0].message.tool_calls[0]
print(tool_call.function.name)     # get_weather
print(tool_call.function.arguments) # {"city": "서울"}  ← JSON 문자열로 반환
```
```python
# Pydantic 구조화 출력
from pydantic import BaseModel, Field
from typing import List
import json

# ← LLM 응답을 담을 Pydantic 모델 정의 (타입 + 제약조건)
class SentimentResult(BaseModel):
    sentiment: str = Field(description="긍정/부정/중립")
    score: float = Field(ge=0, le=1, description="확신도 점수")  # ← 0~1 범위 제한
    keywords: List[str] = Field(description="핵심 키워드")

response = client.chat.completions.create(
    model="gpt-4",
    messages=[{
        "role": "user",
        "content": "이 리뷰 분석: '배송 빠르고 품질 좋아요'",
    }],
    response_format={"type": "json_object"},  # ← JSON 출력 강제
)
# ← model_validate_json(): JSON 문자열을 Pydantic 모델로 변환 + 검증
result = SentimentResult.model_validate_json(
    response.choices[0].message.content
)
print(result.model_dump())  # ← Pydantic v2: dict로 직렬화

# 출력 검증
if result.score < 0.5:
    raise ValueError("낮은 확신도")  # ← 비즈니스 로직에 따른 추가 검증
```
```python
# instructor 라이브러리로 더 간결하게
import instructor

# ← from_openai(): OpenAI 클라이언트를 instructor로 래핑
client = instructor.from_openai(OpenAI())

# ← response_model만 지정하면 instructor가 내부에서 Function Calling + Pydantic 검증 자동 처리
response = client.chat.completions.create(
    model="gpt-4",
    response_model=SentimentResult,  # ← Pydantic 모델을 직접 지정
    messages=[{"role": "user", "content": "배송이 너무 느려요"}],
)
# ← response는 이미 검증된 SentimentResult 인스턴스
print(response.sentiment, response.score)
```

## 주요 패턴
- Few-shot 예시 제공: 2~4개의 예시로 원하는 출력 포맷을 학습시킴
- Chain-of-Thought: "단계별로 생각해보자" 유도문으로 추론 과정을 명시적으로 드러냄
- Function Calling + Pydantic: 함수 파라미터를 Pydantic 모델로 정의하고 자동 검증
- JSON Schema 강제: `response_format={type: "json_object"}`으로 JSON 출력을 보장 (단, 프롬프트에 "JSON" 언급 필수)
- 출력 재시도 루프: 파싱 실패 시 피드백을 포함해 재요청하는 패턴

## 주의사항
- JSON 스키마가 너무 복잡하면 모델이 스키마를 무시하고 자유 형식으로 응답할 수 있음
- Few-shot 예시가 편향되면 출력이 예시 방향으로 쏠리는 현상 발생
- Function Calling 시 tool_choice="auto"가 예상치 못한 도구를 호출할 수 있음 -> `tool_choice={"type": "function", "function": {"name": "my_func"}}`로 강제 지정 가능
- Pydantic 검증이 실패할 경우를 대비한 재시도 로직이 반드시 필요함
- `response_format={"type": "json_object"}` 사용 시 프롬프트에 "JSON"이라는 단어를 반드시 포함해야 모델이 JSON을 출력함 (OpenAI 공식 요구사항)
- instructor 라이브러리는 내부적으로 재시도 로직을 포함하지만, `max_retries` 파라미터로 재시도 횟수 제어 가능
- `tool_calls`는 모델 정책에 따라 빈 배열이거나 None일 수 있으므로 항상 null 체크 필요
