# Machine Learning

## Official Documentation
- [Scikit-Learn User Guide](https://scikit-learn.org/stable/user_guide.html)

## 핵심 개념
> Scikit-Learn은 일관된 `Estimator` 인터페이스(`.fit()`/`.predict()`/`.transform()`)로 분류, 회귀, 클러스터링, 차원 축소 등 고전 머신러닝 알고리즘을 제공한다. `Pipeline`으로 전처리(스케일링, 인코딩, 결측치 처리)와 모델을 하나의 객체로 연결하여 data leakage를 방지하고, `ColumnTransformer`로 컬럼별 상이한 전처리를 적용한다. `train_test_split()`과 `cross_val_score()`로 데이터를 분할·검증하며, `GridSearchCV`/`RandomizedSearchCV`로 하이퍼파라미터를 자동 탐색한다. XGBoost와 LightGBM은 Gradient Boosting Decision Tree(GBDT) 프레임워크로, 테이블 데이터에서 딥러닝을 능가하는 성능을 내며 `early_stopping_rounds`로 과적합을 방지한다.

## 학습 목표
- Scikit-Learn의 `fit()`/`predict()` 패턴으로 분류/회귀 모델을 훈련할 수 있다.
- `Pipeline`으로 전처리와 모델 학습을 통합할 수 있다.
- `train_test_split()`과 교차검증으로 모델을 평가할 수 있다.
- XGBoost와 LightGBM을 사용한 Gradient Boosting 모델을 구성할 수 있다.

## 예제 코드
```python
import numpy as np  # ← 수치 연산용 NumPy
from sklearn.datasets import make_classification  # ← 분류용 가상 데이터 생성기
from sklearn.model_selection import train_test_split, cross_val_score  # ← 데이터 분할·교차검증
from sklearn.preprocessing import StandardScaler  # ← 표준화 전처리기 (평균0, 분산1)
from sklearn.pipeline import Pipeline  # ← 전처리+모델을 하나로 연결하는 파이프라인
from sklearn.linear_model import LogisticRegression  # ← 로지스틱 회귀 분류기
from sklearn.ensemble import RandomForestClassifier  # ← 랜덤 포레스트 분류기
from sklearn.metrics import accuracy_score, classification_report  # ← 평가 지표
import xgboost as xgb  # ← XGBoost: 고성능 GBDT 라이브러리
import lightgbm as lgb  # ← LightGBM: Microsoft의 고속 GBDT 라이브러리

# --- 데이터 생성 ---
X, y = make_classification(  # ← 분류용 합성 데이터 생성
    n_samples=1000, n_features=20,  # ← 1000개 샘플, 20개 특성
    n_informative=10,  # ← 실제로 분류에 기여하는 특성 10개
    n_redundant=5,  # ← informative 특성의 선형 결합으로 생성된 중복 특성 5개
    random_state=42  # ← 재현 가능한 결과를 위한 시드 고정
)

# --- train_test_split ---
X_train, X_test, y_train, y_test = train_test_split(  # ← 데이터를 훈련·테스트로 분할
    X, y, test_size=0.2,  # ← 20%를 테스트 셋으로 분리
    random_state=42,  # ← 분할 시드 고정
    stratify=y  # ← 클래스 비율을 훈련·테스트 셋에서 동일하게 유지 (불균형 대응)
)

# --- Pipeline (Scaler + LogisticRegression) ---
pipe = Pipeline([  # ← 전처리 -> 모델 순서로 단계 정의
    ("scaler", StandardScaler()),  # ← 1단계: 특성 표준화 (평균 0, 표준편차 1)
    ("clf", LogisticRegression(max_iter=1000)),  # ← 2단계: 로지스틱 회귀 (최대 반복 1000)
])
pipe.fit(X_train, y_train)  # ← 전처리 학습 + 모델 학습을 한 번에 수행
print(f"Pipeline Accuracy: {pipe.score(X_test, y_test):.4f}")  # ← 테스트 정확도 출력

# --- 교차검증 (RandomForest) ---
rf = RandomForestClassifier(n_estimators=100, random_state=42)  # ← 100개 의사결정나무 앙상블
cv_scores = cross_val_score(rf, X, y, cv=5, scoring="accuracy")  # ← 5-Fold 교차검증 수행
print(f"RF CV Scores: {cv_scores}")  # ← 각 폴드별 점수 출력
print(f"RF CV Mean: {cv_scores.mean():.4f} (+/- {cv_scores.std():.4f})")  # ← 평균 ± 표준편차

# --- 전체 평가 ---
rf.fit(X_train, y_train)  # ← 전체 훈련 데이터로 최종 학습
y_pred = rf.predict(X_test)  # ← 테스트 데이터 예측
print(f"\nRF Test Accuracy: {accuracy_score(y_test, y_pred):.4f}")  # ← 예측 정확도
print(classification_report(y_test, y_pred, target_names=["Class 0", "Class 1"]))  # ← 분류 보고서

# --- XGBoost ---
xgb_clf = xgb.XGBClassifier(  # ← XGBoost 분류기 생성
    n_estimators=100,  # ← 트리 개수 (반복 횟수)
    learning_rate=0.1,  # ← 학습률 (각 트리의 기여도 가중치)
    max_depth=5,  # ← 각 트리의 최대 깊이 (과적합 방지)
    random_state=42  # ← 재현용 시드
)
xgb_clf.fit(X_train, y_train)  # ← XGBoost 학습
print(f"XGBoost Accuracy: {accuracy_score(y_test, xgb_clf.predict(X_test)):.4f}")

# --- LightGBM ---
lgb_clf = lgb.LGBMClassifier(  # ← LightGBM 분류기 생성
    n_estimators=100,  # ← 트리 개수
    learning_rate=0.1,  # ← 학습률
    max_depth=5,  # ← 최대 깊이
    random_state=42,  # ← 재현용 시드
    verbose=-1  # ← 학습 로그 출력 억제 (-1 = 무음)
)
lgb_clf.fit(X_train, y_train)  # ← LightGBM 학습
print(f"LightGBM Accuracy: {accuracy_score(y_test, lgb_clf.predict(X_test)):.4f}")
```

## 주요 패턴
- `Pipeline`: `(name, transformer/estimator)` 튜플 리스트로 구성하여 전처리 누수를 방지하고 재사용성을 높인다.
- `train_test_split(stratify=y)`: 클래스 불균형 데이터에서 타겟 분포를 유지하며 분할한다.
- `cross_val_score`: K-Fold 교차검증으로 단일 train/test보다 안정적인 성능 추정을 제공한다.
- XGBoost/LightGBM: `early_stopping_rounds`로 검증 손실이 개선되지 않으면 학습을 조기 중단한다.
- `classification_report`: 정밀도(Precision), 재현율(Recall), F1-score를 한눈에 보여준다.

## 주의사항
- 학습 전에 반드시 `train_test_split`으로 데이터를 분리하고, 테스트 셋 정보가 훈련에 유입되지 않도록 한다.
- `Pipeline` 내에서 `StandardScaler`는 `fit()` 시 테스트 데이터를 보지 않으므로 data leakage를 방지한다.
- XGBoost/LightGBM은 `categorical_feature` 파라미터를 명시하지 않으면 범주형 변수를 자동 감지하지 못할 수 있다.
- `cross_val_score`의 기본 scoring은 분류기일 때 `accuracy`이므로, 불균형 데이터에서는 `f1`이나 `roc_auc`를 지정한다.
- LightGBM의 `verbose=-1`로 과도한 훈련 로그 출력을 억제할 수 있다.
- `train_test_split`에서 `random_state`를 고정하지 않으면 매 실행마다 결과가 달라져 디버깅이 어려워진다.
- XGBoost의 `n_estimators`가 너무 크면 과적합 위험이 있으므로, `early_stopping_rounds`와 검증 셋(`eval_set`)을 함께 사용한다.
