# Async

## Official Documentation
- [asyncio — Asynchronous I/O](https://docs.python.org/3/library/asyncio.html)

## 핵심 개념
> `asyncio`는 단일 스레드에서 협력적 멀티태스킹을 구현하는 비동기 I/O 라이브러리다. 이벤트 루프(Event Loop)가 코루틴을 스케줄링하며, `await` 지점에서 실행 권한을 자발적으로 양보하여 다른 코루틴이 실행된다 -> 선점형 멀티스레딩과 달리 경쟁 상태가 근본적으로 적다. `async`/`await` 문법으로 코루틴을 정의하고, `asyncio.run()`으로 이벤트 루프를 실행한다. `gather()`와 `TaskGroup`(3.11+)으로 여러 코루틴을 동시에 실행하며, `asyncio.Queue`로 생산자-소비자 패턴을 구현한다. CPU 바운드 작업은 `Threading`(GIL 제약이 적은 I/O 혼합 작업)이나 `Multiprocessing`(순수 병렬 연산)으로 처리하고, 공유 자원 접근은 `Lock`으로 동기화한다.

## 학습 목표
- `async`/`await`로 비동기 함수를 작성하고 `asyncio.run()`으로 실행할 수 있다.
- `gather()`와 `TaskGroup`으로 여러 비동기 작업을 동시에 처리할 수 있다.
- `asyncio.Queue`를 활용한 생산자-소비자 패턴을 구현할 수 있다.
- `Threading`, `Multiprocessing`, `Lock`으로 동시성 이슈를 해결할 수 있다.

## 예제 코드
```python
import asyncio  # ← 비동기 I/O 코어 라이브러리
import time  # ← 동기 sleep (스레드 예제에서만 사용)
import threading  # ← 멀티스레딩 라이브러리 (I/O 병렬 처리용)
import multiprocessing  # ← 멀티프로세싱 라이브러리 (CPU 병렬 처리용)
from typing import AsyncIterator  # ← 비동기 이터레이터 타입 힌트

# --- async/await + gather ---
async def fetch(id: int, delay: float) -> str:  # ← 비동기 함수 정의 (async def)
    await asyncio.sleep(delay)  # ← 비동기 sleep: 기다리는 동안 다른 코루틴이 실행됨
    return f"Data-{id}"  # ← 지연 후 결과 문자열 반환

async def gather_example():  # ← 여러 코루틴을 동시에 실행하는 예제
    results = await asyncio.gather(  # ← 모든 코루틴 완료까지 대기 (병렬 실행)
        fetch(1, 1.0),  # ← 1초 지연 작업 -> 제일 오래 걸림
        fetch(2, 0.5),  # ← 0.5초 지연 작업
        fetch(3, 0.2),  # ← 0.2초 지연 작업
    )  # ← 전체 소요 ≈ 1.0초 (가장 긴 작업 기준, 병렬이므로)
    print(f"gather results: {results}")  # ← [Data-1, Data-2, Data-3] 순서 보장

# --- TaskGroup (Python 3.11+) ---
async def taskgroup_example():  # ← TaskGroup: 예외 발생 시 나머지 태스크 자동 취소
    async with asyncio.TaskGroup() as tg:  # ← TaskGroup 컨텍스트 진입
        t1 = tg.create_task(fetch(10, 0.3))  # ← 태스크 생성 및 그룹에 등록
        t2 = tg.create_task(fetch(11, 0.1))  # ← 두 번째 태스크
    # ← 컨텍스트 종료 시 모든 태스크 완료를 자동 대기
    print(f"TaskGroup results: {t1.result()}, {t2.result()}")  # ← 각 태스크 결과 조회

# --- Queue (producer-consumer) ---
async def producer(q: asyncio.Queue[int], n: int):  # ← 생산자: n개 항목을 큐에 넣음
    for i in range(n):  # ← 0부터 n-1까지 반복
        await asyncio.sleep(0.1)  # ← 생산 간격 시뮬레이션 (비동기)
        await q.put(i)  # ← 큐에 항목 추가 (put도 await 필요)
        print(f"  Produced: {i}")  # ← 생산 로그 출력

async def consumer(q: asyncio.Queue[int], name: str):  # ← 소비자: 큐에서 항목을 꺼내 처리
    while True:  # ← 무한 루프: 종료 신호(None)를 받을 때까지 계속
        item = await q.get()  # ← 큐에서 항목 가져오기 (빈 큐면 대기)
        if item is None:  # ← None = 종료 신호
            q.task_done()  # ← 종료 신호도 작업 완료로 표시
            break  # ← 루프 탈출
        print(f"  [{name}] Consumed: {item}")  # ← 소비 로그 출력
        await asyncio.sleep(0.2)  # ← 처리 시간 시뮬레이션
        q.task_done()  # ← 작업 완료 알림 (q.join()과 쌍을 이룸)

async def queue_example():
    q: asyncio.Queue[int] = asyncio.Queue()  # ← 정수형 비동기 큐 생성
    async with asyncio.TaskGroup() as tg:  # ← 생산자·소비자를 한 그룹에서 관리
        tg.create_task(producer(q, 5))  # ← 생산자 태스크: 0~4까지 5개 생산
        tg.create_task(consumer(q, "A"))  # ← 소비자 A
        tg.create_task(consumer(q, "B"))  # ← 소비자 B
        tg.create_task(asyncio.sleep(1.1))  # ← 1.1초 후 종료 신호 전송 준비
        for _ in range(2):  # ← 소비자 2명에게 종료 신호 전송
            await q.put(None)  # ← None을 큐에 넣어 각 소비자 종료 유도

# --- Threading ---
lock = threading.Lock()  # ← 스레드 락 생성: 한 번에 한 스레드만 임계 영역 진입
shared_counter = 0  # ← 여러 스레드가 공유할 전역 카운터

def thread_task():  # ← 각 스레드가 실행할 작업 함수
    global shared_counter  # ← 전역 변수 사용 선언
    with lock:  # ← 락 획득 -> 임계 영역 진입 (다른 스레드 대기)
        local = shared_counter  # ← 현재 값을 지역 변수에 복사
        time.sleep(0.01)  # ← 의도적 지연 (경쟁 상태 재현 용도, 실제로는 불필요)
        shared_counter = local + 1  # ← 복사본 + 1을 다시 저장 -> 원자적 증가

def threading_example():
    threads = [threading.Thread(target=thread_task) for _ in range(10)]  # ← 스레드 10개 생성
    for t in threads:
        t.start()  # ← 각 스레드 시작
    for t in threads:
        t.join()  # ← 모든 스레드가 종료될 때까지 대기
    print(f"Threading counter: {shared_counter}")  # ← 최종 카운터 -> 락 덕분에 10

# --- Multiprocessing ---
def cpu_task(n: int) -> int:  # ← CPU 집약 연산: 별도 프로세스에서 GIL 없이 실행됨
    return sum(i * i for i in range(n))  # ← 제곱의 합 계산 (제너레이터로 메모리 절약)

def multiprocessing_example():
    with multiprocessing.Pool(processes=4) as pool:  # ← 4개 프로세스 풀 생성
        results = pool.map(cpu_task, [10_000, 20_000, 30_000])  # ← 입력별 병렬 실행
    # ← with 블록 종료 시 풀 자동 정리
    print(f"Multiprocessing results: {results}")  # ← [합1, 합2, 합3] 출력

# --- 메인 ---
async def main():  # ← 최상위 진입점 코루틴
    await gather_example()  # ← gather 예제 실행
    await taskgroup_example()  # ← TaskGroup 예제 실행
    await queue_example()  # ← Queue 예제 실행
    threading_example()  # ← 동기 함수 호출 (이 순간 이벤트 루프 블로킹)
    multiprocessing_example()  # ← 동기 함수 호출 (이 순간 이벤트 루프 블로킹)

if __name__ == "__main__":  # ← 직접 실행 시에만 main 실행
    asyncio.run(main())  # ← 새 이벤트 루프 생성 -> main() 실행 -> 종료
```

## 주요 패턴
- `asyncio.gather(*coros)`: 여러 코루틴을 동시에 실행하고 모든 결과를 리스트로 반환한다.
- `TaskGroup`: 예외 발생 시 나머지 태스크를 자동 취소하여 리소스 누수를 방지한다.
- `asyncio.Queue`: `put()`/`get()`이 모두 awaitable이므로 논블로킹 생산자-소비자 구현이 가능하다.
- Threading + Lock: I/O 바운드 작업에 유용하며 `threading.Lock()`으로 경쟁 상태를 방지한다.
- Multiprocessing Pool: CPU 바운드 병렬 작업에 적합하며 GIL 제약 없이 멀티코어를 활용한다.

## 주의사항
- `async` 함수 내에서 `time.sleep()` 대신 `await asyncio.sleep()`을 사용해야 이벤트 루프가 블로킹되지 않는다.
- `asyncio.run()`으로 생성된 이벤트 루프는 모든 태스크 완료 후 종료되며, 중첩 호출할 수 없다 (이미 실행 중인 이벤트 루프가 있으면 `RuntimeError`).
- `threading.Lock()`은 코루틴과 호환되지 않으므로 비동기 컨텍스트에서는 `asyncio.Lock()`을 사용한다.
- Multiprocessing에서 공유 객체는 pickle 직렬화 가능해야 하며, 람다나 클로저 전달에 주의한다.
- `TaskGroup` 사용 시 `tg.create_task()`로 생성한 태스크만 취소 보호를 받는다.
- 코루틴을 `await` 없이 호출하면 코루틴 객체만 생성되고 실행되지 않으므로 주의한다 (`RuntimeWarning: coroutine was never awaited`).
- `asyncio.create_task()`로 생성한 태스크가 예외를 발생시키면 `gather()` 시점까지 전파되지 않으므로, 개별 태스크의 `result()`나 `exception()`을 명시적으로 확인한다.
