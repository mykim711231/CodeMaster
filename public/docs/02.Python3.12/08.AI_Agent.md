# AI Agent

## Official Documentation
- [LangGraph](https://langchain-ai.github.io/langgraph/)

## 핵심 개념
> AI Agent는 LLM이 스스로 사고(Reasoning)하고, 도구를 호출(Tool Calling)하며, 계획(Planning)을 세우고 행동(Acting)하는 자율 시스템이다. 단순한 Q&A를 넘어, Agent는 필요 시 외부 API·계산기·데이터베이스를 호출하고, 그 결과를 바탕으로 다음 행동을 결정한다. Tool/Function Calling은 LLM이 함수 시그니처를 이해하고 적절한 인자를 생성하여 외부 함수를 호출하게 하는 메커니즘이다. Memory는 대화 맥락을 저장·요약·검색하여 장기적 상호작용을 가능하게 한다. ReAct(Reasoning + Acting)는 Thought -> Action -> Observation 순환 패턴으로, Reflection은 자신의 출력을 스스로 평가·개선하는 메타인지 패턴이다. Multi-Agent는 여러 에이전트가 Supervisor-Worker, Debate, Collaborative 등 구조로 협업하는 설계다.

## 학습 목표
- Tool/Function Calling으로 LLM이 외부 함수를 호출하는 파이프라인 구현
- Memory 기법으로 대화 이력 관리 및 컨텍스트 유지
- Planning 전략으로 작업 분해 및 Plan-and-Execute 사이클 설계
- ReAct 패턴(Thought -> Action -> Observation)으로 추론-행동 루프 구성
- Reflection 및 Self-Critique를 통한 출력 품질 개선
- LangGraph를 활용한 Multi-Agent 협업 및 상태 기반 워크플로우 설계

## 예제 코드
```python
from langgraph.graph import StateGraph, END  # ← 상태 기반 그래프 워크플로우 엔진
from typing import TypedDict, Annotated  # ← 타입 힌트 (TypedDict=구조화된 딕셔너리 타입)
import operator  # ← operator.add: 리스트 결합용

# 1. Tool 정의 (Function Calling)
from langchain.tools import tool  # ← @tool 데코레이터: 함수를 LLM 호출 가능한 도구로 변환

@tool  # ← 이 데코레이터로 함수의 docstring이 LLM에게 도구 설명으로 전달됨
def calculator(expression: str) -> float:  # ← LLM이 호출할 계산기 도구
    """Evaluate a mathematical expression."""  # ← docstring이 LLM에게 함수 용도 설명
    return eval(expression)  # ← 수식 문자열을 평가하여 결과 반환 (보안상 eval은 예제용)

tools = [calculator]  # ← 에이전트가 사용할 수 있는 도구 목록

# 2. LangGraph로 ReAct Agent 구성
class AgentState(TypedDict):  # ← 에이전트의 상태 구조 정의 (TypedDict = 타입 힌트가 있는 dict)
    messages: Annotated[list, operator.add]  # ← 대화 메시지 리스트 (Annotated로 누적 방식 지정: add)
    next_action: str  # ← 다음에 실행할 노드 이름 ("agent" 또는 "tools")

def agent(state: AgentState):  # ← Agent 노드: LLM이 다음 행동을 결정
    # LLM이 Tool 호출 또는 최종 응답 중 하나를 선택하여 반환
    response = llm_with_tools.invoke(state["messages"])  # ← LLM 호출 (Tool 바인딩된 모델)
    return {"messages": [response], "next_action": "continue"}  # ← 응답을 메시지 목록에 추가

def tool_executor(state: AgentState):  # ← Tool 실행 노드: LLM이 요청한 도구를 실제 실행
    last_message = state["messages"][-1]  # ← 마지막 메시지 (LLM의 Tool 호출 응답)
    for tool_call in last_message.tool_calls:  # ← LLM이 요청한 모든 도구 호출 반복
        result = execute_tool(tool_call)  # ← 실제 Tool 실행 (별도 구현 필요)
        state["messages"].append(result)  # ← Tool 실행 결과를 메시지에 추가
    return {"messages": state["messages"], "next_action": "agent"}  # ← 다시 Agent 노드로

def should_continue(state: AgentState) -> str:  # ← 조건부 라우팅: 다음 노드 결정
    last_message = state["messages"][-1]  # ← 마지막 메시지 확인
    if hasattr(last_message, "tool_calls") and last_message.tool_calls:  # ← Tool 호출이 있으면
        return "tools"  # ← Tool 실행 노드로 분기
    return END  # ← Tool 호출 없으면 그래프 종료 (최종 응답)

# 3. 그래프 구성 및 컴파일
graph = StateGraph(AgentState)  # ← 상태 타입을 지정하여 그래프 생성
graph.add_node("agent", agent)  # ← "agent" 노드 등록
graph.add_node("tools", tool_executor)  # ← "tools" 노드 등록
graph.set_entry_point("agent")  # ← 시작점을 agent 노드로 설정
graph.add_conditional_edges(  # ← 조건부 엣지: should_continue 결과에 따라 분기
    "agent",  # ← agent 노드에서 출발
    should_continue,  # ← 라우팅 함수
    {"tools": "tools", END: END}  # ← "tools" 반환 시 tools 노드로, END 반환 시 종료
)
graph.add_edge("tools", "agent")  # ← tools 노드 실행 후 agent로 복귀 (ReAct 루프)
agent_app = graph.compile()  # ← 그래프 컴파일 -> 실행 가능한 애플리케이션

result = agent_app.invoke({"messages": ["Calculate 15 * 25 + 100"]})  # ← 초기 입력으로 실행
print(result)  # ← 최종 상태 출력 (생성된 답변 포함)

# 4. Reflection 패턴 (자기 평가 및 개선)
def reflect(state: AgentState):  # ← Reflection 노드: 이전 출력을 평가하고 개선
    reflection_prompt = f"""  # ← 자기 반성 프롬프트 템플릿
    Review the previous output and identify any errors or improvements:
    {state['messages'][-1]}  # ← 이전 출력을 프롬프트에 삽입
    Provide a refined response.
    """
    refined = llm.invoke(reflection_prompt)  # ← 개선된 응답 생성
    return {"messages": [refined], "next_action": "continue"}  # ← 개선된 응답 반환
```

## 주요 패턴
- Tool Calling: OpenAI function calling, Anthropic tool use 등으로 모델이 도구 선택
- Memory: ConversationBufferMemory, ConversationSummaryMemory, ConversationKGMemory
- ReAct 루프: Thought -> Action -> Observation -> (반복) -> Final Answer
- Plan-and-Execute: Plan(계획 수립) -> Execute(단계별 실행) -> Replan(필요시 재계획)
- Self-Reflection: Self-Critique, Reflexion, LLM-as-Judge로 출력 평가 및 반복 개선
- Multi-Agent: Supervisor-Worker, Debate, Collaborative로 역할 분담 및 협업
- Conditional Edges: 에이전트 상태에 따라 다음 노드를 동적으로 결정 (라우팅)
- Human-in-the-Loop: `interrupt()`로 특정 노드에서 사람 승인 대기

## 주의사항
- 무한 Tool 호출 루프 방지를 위해 최대 반복 횟수(`recursion_limit`) 설정
- Tool 설명과 파라미터 타입 힌트가 부정확하면 LLM이 잘못된 인자로 함수 호출
- Function Calling은 토큰 소비가 크므로 불필요한 도구 호출을 피하도록 프롬프트 설계
- Memory 저장소가 커질수록 컨텍스트 윈도우 초과 위험 -> 요약 및 오래된 메시지 삭제 전략 필요
- Reflection 과정에서 거짓 정보를 보강할 수 있으므로 최종 검증 단계 추가
- Multi-Agent에서 통신 오버헤드와 비용이 Agent 수에 비례해 증가 -> 필요 최소한으로 설계
- `agent_app.invoke()`의 입력 `messages`에는 반드시 최소 1개의 HumanMessage 또는 문자열이 포함되어야 하며, 빈 리스트 전달 시 오류가 발생한다.
- `llm_with_tools`와 `llm`은 별도로 정의해야 하는 객체로, `llm.bind_tools(tools)`로 Tool을 바인딩한 모델과 기본 LLM 인스턴스를 각각 의미한다 (예제에서는 선언 생략).
