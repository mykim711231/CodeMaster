# Data

## 핵심 개념
> NumPy의 `ndarray`는 고정 타입의 다차원 배열로, C 수준에서 벡터화 연산을 수행해 Python 루프보다 수십~수백 배 빠르다. 브로드캐스팅 규칙을 통해 형태가 다른 배열 간 연산을 자동 확장하며, `ufunc`(Universal Function)로 요소별 연산을 일괄 적용한다. Pandas의 `DataFrame`은 라벨링된 2차원 테이블로, SQL과 유사한 `groupby()` 집계, `merge()` 조인, `pivot_table()` 피벗 기능을 제공한다. Matplotlib은 데이터 시각화의 표준 라이브러리로, `pyplot` (MATLAB 스타일)과 객체 지향 API 두 가지 인터페이스를 제공한다. Polars는 Rust로 작성된 고속 DataFrame 라이브러리로, 지연 평가(LazyFrame)를 통해 실행 계획을 최적화한 후 한 번에 실행하여 Pandas 대비 수 배 빠른 성능을 낸다.

## 학습 목표
- NumPy `ndarray`를 생성하고 브로드캐스팅 규칙을 이해할 수 있다.
- Pandas `DataFrame`으로 데이터를 로드하고 `groupby()`와 `merge()`로 분석할 수 있다.
- Matplotlib으로 기본적인 차트를 생성할 수 있다.
- Polars의 기본 DataFrame API를 사용할 수 있다.

## 예제 코드
```python
import numpy as np  # ← NumPy: 고속 수치 연산 라이브러리
import pandas as pd  # ← Pandas: 테이블 데이터 분석 라이브러리
import matplotlib.pyplot as plt  # ← Matplotlib: 데이터 시각화 라이브러리
import polars as pl  # ← Polars: Rust 기반 고속 DataFrame 라이브러리

# --- NumPy ---
arr1 = np.array([[1, 2, 3], [4, 5, 6]])  # ← shape (2,3)인 2차원 배열 생성
arr2 = np.array([10, 20, 30])  # ← shape (3,)인 1차원 배열 생성

# broadcast: (2,3) + (3,) → (3,)이 (2,3)으로 자동 확장되어 연산
broadcasted = arr1 + arr2  # ← arr2가 각 행에 더해짐: [[11,22,33], [14,25,36]]

# universal functions (ufunc): 요소별 연산을 벡터화하여 적용
sqrt_arr = np.sqrt(broadcasted)  # ← 각 요소의 제곱근 계산 (C 수준 루프)

print("NumPy broadcast result:\n", broadcasted)  # ← 브로드캐스팅 결과 출력
print("NumPy sqrt:\n", sqrt_arr)  # ← 제곱근 결과 출력

# --- Pandas ---
df = pd.DataFrame({  # ← 딕셔너리로 DataFrame 생성
    "category": ["A", "A", "B", "B", "C"],  # ← 범주형 컬럼
    "value":   [10, 20, 30, 40, 50],  # ← 수치형 컬럼
})
df_extra = pd.DataFrame({  # ← 조인할 추가 정보 DataFrame
    "category": ["A", "B", "D"],  # ← 조인 키 컬럼
    "name":     ["Alpha", "Beta", "Delta"],  # ← 매칭할 이름 정보
})

# groupby: category별로 value의 sum과 mean을 집계 (split-apply-combine)
grouped = df.groupby("category")["value"].agg(["sum", "mean"])  # ← 다중 집계

# merge: category 컬럼을 기준으로 inner join (SQL의 JOIN과 동일)
merged = df.merge(df_extra, on="category", how="inner")  # ← A, B만 매칭 (D는 제외)

print("\nPandas groupby:\n", grouped)  # ← 그룹별 통계 출력
print("\nPandas merge:\n", merged)  # ← 조인 결과 출력

# --- Matplotlib ---
x = np.linspace(0, 2 * np.pi, 100)  # ← 0~2π 범위의 균등 간격 점 100개 생성
y_sin = np.sin(x)  # ← x에 대한 sin 값 벡터 계산
y_cos = np.cos(x)  # ← x에 대한 cos 값 벡터 계산

plt.figure(figsize=(8, 4))  # ← 그림 크기 설정 (가로 8, 세로 4 인치)
plt.plot(x, y_sin, label="sin(x)")  # ← sin 곡선 그리기 + 범례 라벨
plt.plot(x, y_cos, label="cos(x)")  # ← cos 곡선 그리기 + 범례 라벨
plt.title("Sine and Cosine")  # ← 그래프 제목
plt.xlabel("x")  # ← x축 라벨
plt.ylabel("y")  # ← y축 라벨
plt.legend()  # ← 범례 표시
plt.grid(True)  # ← 격자선 활성화
plt.tight_layout()  # ← 레이아웃 자동 조정 (라벨 잘림 방지)
plt.savefig("sine_cosine.png")  # ← 현재 그림을 PNG 파일로 저장

# --- Polars ---
df_pl = pl.DataFrame({  # ← Polars DataFrame 생성 (Pandas와 유사한 API)
    "name": ["Alice", "Bob", "Charlie"],  # ← 이름 컬럼
    "age":  [25, 30, 35],  # ← 나이 컬럼
    "score":[88.5, 92.0, 79.5],  # ← 점수 컬럼
})

# filter + select: 나이가 28 초과인 행만 선택 → name, score 컬럼만 추출
filtered = df_pl.filter(pl.col("age") > 28).select(["name", "score"])

# lazy evaluation: 실행 계획을 최적화한 후 .collect()로 한 번에 실행
lazy_result = (
    df_pl.lazy()  # ← LazyFrame으로 변환 (즉시 실행 안 함)
    .filter(pl.col("score") > 80)  # ← 점수 80 초과 필터 (계획만 기록)
    .group_by("age")  # ← age별 그룹화 (Polars 1.0+ API, 구버전은 groupby)
    .agg(pl.col("score").mean())  # ← 각 그룹의 score 평균 집계
    .collect()  # ← 여기서 전체 계획을 한 번에 실행
)

print("\nPolars filtered:\n", filtered)  # ← 필터링 결과 출력
print("\nPolars lazy result:\n", lazy_result)  # ← 지연 평가 결과 출력
```

## 주요 패턴
- NumPy broadcast: 배열 차원을 우측 정렬하여 호환 가능한 차원이면 자동 확장된다.
- Pandas groupby-agg: `split-apply-combine` 전략으로 카테고리별 통계를 계산한다.
- Pandas merge: `on`, `how`(inner/left/right/outer) 파라미터로 SQL 조인과 유사하게 결합한다.
- Matplotlib OOP API: `fig, ax = plt.subplots()`로 서브플롯을 객체 지향적으로 다룬다.
- Polars lazy: `.lazy().pipe1().pipe2().collect()`로 실행 계획을 최적화한 후 한 번에 실행한다.

## 주의사항
- NumPy에서 `shape`가 다른 배열을 연산할 때 브로드캐스팅이 의도치 않게 발생할 수 있으므로 `arr.shape`을 항상 확인한다.
- Pandas DataFrame에서 `inplace=True`는 권장되지 않으며(deprecated 예정), 항상 새 객체를 반환하는 메서드 체이닝을 사용한다.
- Matplotlib은 기본적으로 GUI 백엔드를 사용하므로, 서버 환경에서는 `matplotlib.use("Agg")`를 먼저 호출한다.
- Polars는 Pandas와 API가 유사하지만 미묘하게 다르므로(예: `select` vs `loc`, `group_by` vs `groupby`) 공식 문서를 참조한다.
- 대용량 데이터를 Pandas `merge()` 할 때 카테시안 곱이 발생하지 않도록 조인 키의 유니크 여부를 확인한다.
- NumPy `np.sqrt()`는 음수 입력 시 `nan`을 반환하고 경고를 발생시키므로, 복소수 연산이 필요하면 `np.sqrt(x + 0j)` 또는 `np.emath.sqrt()`를 사용한다.
- Matplotlib에서 `plt.savefig()` 후에는 `plt.close()`로 메모리를 해제해야 반복적인 그래프 생성 시 메모리 누수를 방지할 수 있다.
