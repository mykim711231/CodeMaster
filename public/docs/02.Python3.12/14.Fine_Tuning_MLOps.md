# Fine-Tuning & MLOps
## Official Documentation
- [PEFT 공식 문서](https://huggingface.co/docs/peft/index)
- [MLflow](https://mlflow.org/docs/latest/)
- [DVC](https://dvc.org/doc)
- [Weights & Biases](https://docs.wandb.ai/)

## 핵심 개념
> Fine-tuning은 사전 학습된 LLM을 특정 태스크에 맞게 미세 조정하는 과정이다. Full Fine-tuning은 모델의 모든 파라미터를 업데이트하여 GPU 메모리가 수백 GB 필요하지만, PEFT(Parameter-Efficient Fine-Tuning) 기법은 극히 일부 파라미터만 학습한다. LoRA(Low-Rank Adaptation)는 가중치 행렬 W를 W + ΔW로 분해하고, ΔW를 저랭크 행렬 A×B로 근사하여 전체 파라미터의 1% 미만만 학습한다. QLoRA는 여기에 4-bit NormalFloat(NF4) 양자화, Double Quantization, Paged Optimizer를 결합해 단일 24GB GPU에서도 70B 모델의 Fine-tuning을 가능하게 한다. MLflow는 실험 추적(log_param/log_metric/log_model)과 모델 레지스트리(스테이징·프로덕션 이력 관리)를 제공한다. DVC(Data Version Control)는 Git과 유사한 방식으로 대용량 데이터셋·모델 파일의 버전을 관리한다. W&B(Weights & Biases)는 학습 메트릭의 실시간 시각화와 하이퍼파라미터 스윕(Sweep)을 통한 자동 최적화를 지원한다.

## 학습 목표
- PEFT 라이브러리로 LoRA/QLoRA Fine-tuning 구현하기
- LoRA rank, alpha 등 하이퍼파라미터가 학습에 미치는 영향 이해하기
- MLflow로 실험 메트릭(log_metric)과 모델(log_model) 기록하기
- DVC로 학습 데이터셋 버전 관리 파이프라인 구축하기
- W&B로 학습 곡선 시각화 및 실험 비교하기

## 예제 코드
```python
# QLoRA Fine-tuning
import torch
from transformers import (
    AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig,
    TrainingArguments
)
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training
from trl import SFTTrainer
from datasets import load_dataset

# ← 4-bit 양자화 설정: 모델을 4-bit로 압축해 GPU 메모리 사용량을 1/4로 감소
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",  # ← NormalFloat4: 4-bit 양자화 중 최고 품질
    bnb_4bit_compute_dtype=torch.bfloat16,  # ← 연산은 bfloat16 정밀도로 수행
)

# ← 양자화된 모델 로드 (device_map="auto": 사용 가능한 GPU에 자동 분산)
model = AutoModelForCausalLM.from_pretrained(
    "mistralai/Mistral-7B-v0.1",
    quantization_config=bnb_config,
    device_map="auto",
)
# ← k-bit 학습을 위한 전처리 (양자화된 레이어의 가중치 정규화 등)
model = prepare_model_for_kbit_training(model)

# ← LoRA 설정: 원본 가중치는 동결하고 저랭크 어댑터만 학습
lora_config = LoraConfig(
    r=16,                                      # ← 저랭크 차원 (클수록 표현력↑, 메모리↑)
    lora_alpha=32,                             # ← 스케일링 팩터 (alpha/r 비율로 학습률 결정)
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],  # ← 어텐션 레이어에 LoRA 적용
    lora_dropout=0.05,
    bias="none",                               # ← bias는 학습하지 않음
    task_type="CAUSAL_LM",
)
model = get_peft_model(model, lora_config)  # ← 원본 모델에 LoRA 어댑터 부착
model.print_trainable_parameters()  # 0.2% 정도만 학습 가능 → 전체 파라미터의 0.2%만 업데이트

# ← SFTTrainer: Supervised Fine-Tuning을 위한 Hugging Face Trainer 래퍼
trainer = SFTTrainer(
    model=model,
    args=TrainingArguments(
        output_dir="./qlora-output",
        per_device_train_batch_size=4,          # ← GPU당 배치 크기
        gradient_accumulation_steps=4,          # ← 4스텝씩 누적 후 가중치 업데이트 (실제 배치=16)
        learning_rate=2e-4,
        logging_steps=10,
        save_steps=100,
        num_train_epochs=3,
    ),
    train_dataset=load_dataset("my_dataset", split="train"),
)
trainer.train()
# ← 어댑터만 저장 (원본 모델은 유지) → 용량 작고 여러 태스크에 교체 사용 가능
model.save_pretrained("./qlora-adapter")
```
```python
# MLflow 실험 추적
import mlflow

# ← 실험 이름 설정 (동일 실험명으로 run을 그룹화)
mlflow.set_experiment("llm-finetuning")
# ← 하나의 run 시작: 모든 파라미터/메트릭이 이 run에 기록됨
mlflow.start_run(run_name="mistral-qlora-v1")

# ← log_param: 하이퍼파라미터 기록 → 재현과 비교에 필수
mlflow.log_param("model", "Mistral-7B-v0.1")
mlflow.log_param("lora_r", 16)
mlflow.log_param("lora_alpha", 32)
mlflow.log_param("learning_rate", 2e-4)

# ← log_metric: 학습 중 메트릭을 step과 함께 기록 → 학습 곡선 시각화 가능
for epoch in range(3):
    train_loss = 0.5 / (epoch + 1)
    val_loss = 0.6 / (epoch + 1)
    mlflow.log_metric("train_loss", train_loss, step=epoch)
    mlflow.log_metric("val_loss", val_loss, step=epoch)

# ← log_model: 학습 완료된 모델을 MLflow Model Registry에 등록
mlflow.log_model(model, "model")
mlflow.end_run()  # ← run 종료 (필수)
```
```python
# W&B 연동
import wandb

# ← wandb.init: 프로젝트 초기화 + config로 하이퍼파라미터 기록
wandb.init(project="llm-finetuning", config={
    "model": "Mistral-7B",
    "lora_r": 16,
    "epochs": 3,
})

# ← wandb.log: 딕셔너리 형태로 메트릭 기록 → W&B 대시보드에서 실시간 그래프 확인
for epoch in range(3):
    metrics = {"train/loss": 0.5/(epoch+1), "val/perplexity": 10+epoch}
    wandb.log(metrics)
wandb.finish()  # ← 실험 종료 (필수)
```

## 주요 패턴
- LoRA 어댑터: 원본 모델은 동결하고 저랭크 행렬만 학습, 어댑터만 교체하여 다중 태스크 대응
- QLoRA 3종 세트: 4-bit NF4 양자화 + Double Quantization + Paged Optimizer로 단일 GPU 학습
- MLflow Tracking: `log_param`, `log_metric`, `log_model`로 재현 가능한 실험 기록
- DVC 파이프라인: `dvc run`으로 데이터 전처리→학습→평가 단계를 DAG로 정의
- W&B Sweep: 하이퍼파라미터 자동 탐색으로 최적 LoRA rank 찾기

## 주의사항
- LoRA target_modules를 잘못 지정하면 학습 효과가 거의 없음 (어텐션 레이어 위주)
- 4-bit 양자화 시 추론과 학습 간 dtype 불일치로 NaN 손실이 발생할 수 있음 → `bnb_4bit_compute_dtype`을 학습 dtype과 일치시킬 것
- MLflow 서버 없이 로컬에서만 실험 추적 시 팀 공유 불가
- DVC 리모트 스토리지 설정 없으면 데이터 유실 위험
- LoRA rank(r)가 너무 작으면(≤4) 과소적합, 너무 크면(≥64) Full Fine-tuning과 메모리 차이가 없어짐 → 8~32 범위에서 시작 권장
- `SFTTrainer`는 내부적으로 `tokenizer`를 자동 로드하지만, 특수 토큰(pad_token 등)은 수동 설정이 필요할 수 있음
- `device_map="auto"`는 accelerate 라이브러리의 추론 기반 분산 전략이므로, 학습 시 예상치 못한 레이어 분산이 발생할 수 있음
