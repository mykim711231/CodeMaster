# Messaging — JMS / AMQP / RabbitMQ
## Official Documentation
- [Spring Boot Messaging](https://docs.spring.io/spring-boot/docs/current/reference/htmlsingle/#messaging)
- [Spring AMQP (RabbitMQ) Reference](https://docs.spring.io/spring-amqp/reference/)
## 핵심 개념
> Spring Boot는 JMS(ActiveMQ 등)와 AMQP(RabbitMQ 등) 메시징 인프라를 자동 설정으로 간편하게 통합한다. `@JmsListener` 나 `@RabbitListener` 어노테이션으로 메시지 리스너를 선언하고, `JmsTemplate` / `RabbitTemplate` 으로 메시지를 발행한다. 비동기 이벤트 기반 아키텍처에서 서비스 간 결합도를 낮추고 탄력성을 높이는 핵심 수단이다.
## 학습 목표
- `@EnableJms` 로 JMS 활성화하고 `@JmsListener` 로 메시지 구독 구현하기
- `@EnableRabbit` / `@RabbitListener` 로 RabbitMQ 큐 리스너 작성하기
- `JmsTemplate`, `RabbitTemplate` 으로 다양한 포맷의 메시지 발행하기
- MessageConverter(Jackson2)* 로 JSON 직렬화/역직렬화 설정하기
- Dead Letter Queue(DLQ) 설정과 실패 메시지 재처리 패턴 적용하기
- Pub/Sub, Point-to-Point, Request-Reply 등 메시징 패턴 실습하기
## 예제 코드
```java
// RabbitConfig.java — RabbitMQ 교환기(Exchange)와 큐(Queue)를 선언해요
import org.springframework.amqp.core.Binding;
import org.springframework.amqp.core.BindingBuilder;
import org.springframework.amqp.core.DirectExchange;
import org.springframework.amqp.core.Queue;
import org.springframework.amqp.core.QueueBuilder;
import org.springframework.amqp.support.converter.Jackson2JsonMessageConverter;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class RabbitConfig {
    public static final String EXCHANGE = "order.exchange";         // ← 교환기 이름 상수
    public static final String QUEUE = "order.created.queue";       // ← 주 큐 이름
    public static final String DLQ = "order.created.dlq";          // ← Dead Letter 큐 이름

    @Bean
    public Jackson2JsonMessageConverter messageConverter() {       // ← 메시지를 JSON으로 직렬화/역직렬화
        return new Jackson2JsonMessageConverter();
    }

    @Bean
    public Queue orderQueue() {                                     // ← 내구성 있는 큐 생성 (서버 재시작해도 유지)
        return QueueBuilder.durable(QUEUE)
            .deadLetterExchange("")                                 // ← 실패 시 DLQ로 보낼 교환기 (기본 "")
            .deadLetterRoutingKey(DLQ)                              // ← DLQ 라우팅 키
            .build();
    }

    @Bean
    public Queue deadLetterQueue() {                                // ← 실패한 메시지를 보관하는 Dead Letter 큐
        return QueueBuilder.durable(DLQ).build();
    }

    @Bean
    public DirectExchange orderExchange() {                         // ← 라우팅 키와 정확히 일치하는 큐로 전달
        return new DirectExchange(EXCHANGE);
    }

    @Bean
    public Binding binding(Queue orderQueue, DirectExchange exchange) {  // ← 큐와 교환기를 라우팅 키로 연결
        return BindingBuilder
            .bind(orderQueue)
            .to(exchange)
            .with("order.created");
    }
}
```
```java
// OrderEventProducer.java — 메시지를 발행하는 생산자(Producer)예요
import org.springframework.amqp.rabbit.core.RabbitTemplate;
import org.springframework.stereotype.Service;

@Service
public class OrderEventProducer {
    private final RabbitTemplate rabbitTemplate;                    // ← RabbitMQ 메시지 발행 템플릿

    public OrderEventProducer(RabbitTemplate rabbitTemplate) {     // ← 생성자 주입
        this.rabbitTemplate = rabbitTemplate;
    }

    public void sendOrderCreated(Order order) {                    // ← 주문 생성 이벤트 발행
        rabbitTemplate.convertAndSend(                              // ← 객체를 JSON으로 변환해 전송
            RabbitConfig.EXCHANGE,                                 // ← 보낼 교환기
            "order.created",                                       // ← 라우팅 키
            order                                                  // ← 전송할 메시지 객체
        );
    }
}
```
```java
// OrderEventHandler.java — 메시지를 소비하는 소비자(Consumer)예요
import org.springframework.amqp.rabbit.annotation.RabbitListener;
import org.springframework.stereotype.Component;

@Component
public class OrderEventHandler {
    @RabbitListener(queues = RabbitConfig.QUEUE)                   // ← 지정된 큐에서 메시지를 수신해요
    public void handleOrderCreated(Order order) {                  // ← Order 객체로 자동 변환되어 들어와요
        System.out.println("Order received: " + order.getId());
    }
}
```
## 주요 패턴
- Publish-Subscribe: Fanout Exchange로 여러 큐에 동시 브로드캐스트
- Work Queues: 한 큐에 여러 컨슈머를 붙여 처리량 분산 (prefetch 설정으로 조절)
- Request-Reply: `@SendTo` 또는 `convertSendAndReceive()` 로 동기적 응답 처리
- Dead Lettering: 실패한 메시지를 DLQ로 이동시켜 별도 분석·재처리
## 주의사항
- `spring.rabbitmq.listener.simple.acknowledge-mode` 기본값은 `auto` — 수동 ACK 시 `MANUAL` 로 변경 필요
- `@RabbitListener` 메서드가 예외를 던지면 기본적으로 메시지가 requeue 되어 무한 재처리 루프에 빠질 수 있음
- 대용량 처리를 위한 `concurrency` / `prefetch` 설정은 컨슈머 수와 메모리 사용량을 고려해 튜닝해야 함
- Connection Factory가 `CachingConnectionFactory`인지 확인: 프로덕션에서는 Chunk 수준으로 채널 풀링 필요
