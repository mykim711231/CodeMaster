## 핵심 개념
> 벡터 저장소는 임베딩 벡터를 저장하고 유사도 검색을 수행하는 데이터베이스다. 텍스트→임베딩 모델→벡터화→저장소 적재의 파이프라인으로 구성되며, 코사인 유사도(Cosine Similarity), 유클리디안 거리(Euclidean), 내적(Inner Product) 등 다양한 거리 함수로 유사도를 측정한다. FAISS는 Meta가 개발한 인메모리 벡터 검색 라이브러리로, HNSW·IVF·PQ 등 다양한 인덱싱 알고리즘을 제공한다. Milvus는 분산형 벡터 DB로 수십억 건 규모의 벡터를 처리한다. Qdrant는 Rust 기반의 고성능 벡터 DB이며, Weaviate는 GraphQL API와 하이브리드 검색을 지원한다. pgvector는 PostgreSQL 확장으로, 기존 RDBMS에 벡터 검색 기능을 추가한다. HNSW(Hierarchical Navigable Small World)는 근사 최근접 이웃(ANN) 검색을 위한 그래프 기반 인덱싱 알고리즘으로, 로그 시간 복잡도로 검색이 가능하다. 메타데이터 필터를 통해 벡터 검색과 메타데이터 조건을 결합한 하이브리드 검색이 가능하다.

## 학습 목표
- pgvector를 사용하여 PostgreSQL에서 벡터 검색 구현하기
- Qdrant, Weaviate, Milvus의 클라이언트 API 사용법 익히기
- FAISS 인덱스 구축 및 유사도 검색 수행하기
- HNSW 알고리즘의 동작 원리와 파라미터 튜닝 이해하기
- 메타데이터 필터를 활용한 하이브리드 검색 구현하기

## 예제 코드
```python
import faiss
import numpy as np

# 128차원 임베딩 벡터 1000개 생성 (실제로는 embedding model 출력값)
dimension = 128
num_vectors = 1000
# ← float32는 FAISS가 요구하는 데이터 타입이에요
vectors = np.random.random((num_vectors, dimension)).astype('float32')

# ← HNSW 인덱스 구축: 그래프 기반 근사 검색 구조
M = 32  # 그래프 연결 수 (클수록 정확도↑, 메모리↑)
index = faiss.IndexHNSWFlat(dimension, M)  # ← Flat: 실제 벡터를 그래프에 저장
index.hnsw.efConstruction = 200  # ← 인덱스 구축 시 탐색 폭 (클수록 정확도↑, 시간↑)
index.add(vectors)  # ← 벡터들을 인덱스에 추가

# ← 검색: query 벡터와 가장 가까운 k=5개 찾기
query = np.random.random((1, dimension)).astype('float32')
index.hnsw.efSearch = 64  # ← 검색 시 탐색 폭 (클수록 정확도↑, 속도↓)
distances, indices = index.search(query, k=5)  # ← 거리와 인덱스 반환
print(f"가장 가까운 5개 인덱스: {indices[0]}")
print(f"거리: {distances[0]}")
```
```python
# pgvector 예제
import psycopg2
import numpy as np

# ← PostgreSQL 연결
conn = psycopg2.connect("dbname=mydb user=user password=pass")
cur = conn.cursor()
cur.execute("CREATE EXTENSION IF NOT EXISTS vector")  # ← pgvector 확장 활성화
cur.execute("""
    CREATE TABLE IF NOT EXISTS documents (
        id SERIAL PRIMARY KEY,
        content TEXT,
        embedding vector(1536)  # ← 1536차원 벡터 컬럼 (OpenAI ada-002 기준)
    )
""")
# ← 검색용 쿼리 벡터 (실제로는 임베딩 모델로 생성)
query_embedding = np.random.random(1536).astype('float32').tolist()
# ← <=> 연산자: 코사인 거리 기반 정렬 (가까운 순)
cur.execute(
    "SELECT content FROM documents "
    "ORDER BY embedding <=> %s LIMIT 5",
    (query_embedding,)
)
results = cur.fetchall()  # ← 검색 결과 가져오기
conn.commit()
cur.close()
conn.close()
```
```python
# Qdrant 예제
from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams, Filter, FieldCondition, MatchValue

client = QdrantClient("localhost", port=6333)  # ← Qdrant 서버 연결 (기본 포트: 6333)
# ← 컬렉션 생성: 벡터 128차원, 코사인 유사도 사용
client.create_collection(
    collection_name="my_collection",
    vectors_config=VectorParams(size=128, distance=Distance.COSINE),
)
# ← 벡터 + 메타데이터(payload)를 함께 저장 (upsert: 없으면 insert, 있으면 update)
client.upsert(
    collection_name="my_collection",
    points=[
        {"id": i, "vector": v.tolist(), "payload": {"source": "web"}}
        for i, v in enumerate(vectors)
    ],
)
# ← 메타데이터 필터(source="web") + 벡터 유사도 검색 = 하이브리드 검색
results = client.search(
    collection_name="my_collection",
    query_vector=query[0].tolist(),
    query_filter=Filter(
        must=[FieldCondition(key="source", match=MatchValue(value="web"))]
    ),
    limit=5,
)
```

## 주요 패턴
- HNSW 인덱싱: 고차원 벡터에서 로그 시간 근사 검색을 위한 그래프 계층 구조
- IVFPQ: 클러스터링으로 검색 범위를 줄이고 PQ(Product Quantization)로 메모리를 압축하는 조합
- 하이브리드 검색: 벡터 유사도와 메타데이터 필터를 결합하여 정밀도 향상
- 배치 임베딩 저장: 대량 임베딩을 청크 단위로 나누어 upsert하여 메모리 관리
- 인덱스 직렬화: 학습된 인덱스를 파일로 저장/로드하여 재학습 방지

## 주의사항
- 벡터 차원이 일치하지 않으면 인덱스 추가 시 오류 발생
- efSearch 값을 너무 낮게 설정하면 검색 정확도가 급격히 저하됨
- FAISS 인덱스는 메모리 기반이므로 프로세스 종료 시 휘발됨 → `faiss.write_index()`로 디스크 저장
- pgvector에서 대량 벡터에 대한 인덱스 생성 시 디스크 공간과 시간 소요가 큼
- pgvector의 `<=>` 연산자는 코사인 거리, `<->`는 유클리디안 거리, `<#>`는 내적(음수)을 반환하므로 용도에 맞는 연산자 선택 필요
- Qdrant 컬렉션 생성 시 `Distance`를 잘못 설정하면 검색 결과의 의미가 완전히 달라짐 (Cosine vs Euclidean vs Dot)
