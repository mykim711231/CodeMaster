# Python Language Basics

## Official Documentation
- [The Python Tutorial](https://docs.python.org/3/tutorial/)

## 핵심 개념
> Python의 핵심 문법 요소들을 다룬다. `def`로 함수를 정의하고, `class`로 객체지향 설계를 하며, `@dataclass`로 데이터 컨테이너를 간결하게 만든다. `typing` 모듈로 타입 힌트를 제공하고, list/dict/set comprehension으로 컬렉션을 선언적으로 생성한다. `with` 문은 `__enter__`와 `__exit__` 매직 메서드(또는 `@contextmanager`)를 통해 파일·소켓·DB 연결 등 리소스의 획득과 해제를 자동화한다. 데코레이터는 `@` 기호로 함수나 클래스를 감싸며, 로깅·인증·캐싱 등 횡단 관심사(Cross-Cutting Concern)를 재사용 가능하게 분리한다. `@dataclass(frozen=True)`는 `__init__`, `__repr__`, `__eq__`를 자동 생성하여 데이터 모델링의 보일러플레이트를 제거한다. `TypeVar`와 `Generic`을 조합하면 타입 안전한 제네릭 컨테이너를 정의할 수 있으며, 이는 mypy 같은 정적 타입 검사기와 함께 사용해야 실효성을 발휘한다.

## 학습 목표
- `def`와 `class`로 기본적인 함수와 클래스를 정의할 수 있다.
- `dataclass`로 보일러플레이트 없는 데이터 클래스를 만들 수 있다.
- Comprehension으로 컬렉션을 간결하게 생성하고 `with`로 컨텍스트 관리자를 활용할 수 있다.
- Decorator를 직접 작성하고 `typing`으로 타입 힌트를 적용할 수 있다.

## 예제 코드
```python
from dataclasses import dataclass  # ← 데이터 클래스 데코레이터 불러오기
from typing import TypeVar, Generic  # ← 제네릭 타입 힌트 도구 불러오기
from functools import wraps  # ← 데코레이터의 메타데이터 보존 도구
from contextlib import contextmanager  # ← 컨텍스트 매니저 제작 도구

# --- typing ---
T = TypeVar("T")  # ← 제네릭 타입 변수 T 선언 (어떤 타입이든 가능)

class Stack(Generic[T]):  # ← T 타입을 받는 제네릭 스택 클래스 정의
    def __init__(self) -> None:  # ← 생성자: 빈 리스트로 초기화
        self._items: list[T] = []  # ← T 타입 항목을 담을 내부 리스트

    def push(self, item: T) -> None:  # ← 항목 추가 메서드, 매개변수 타입은 T
        self._items.append(item)  # ← 리스트 끝에 항목 추가

    def pop(self) -> T:  # ← 항목 꺼내기 메서드, 반환 타입은 T
        return self._items.pop()  # ← 리스트 마지막 항목을 제거하며 반환

# --- dataclass ---
@dataclass(frozen=True)  # ← 불변 데이터 클래스 (frozen=True → 생성 후 필드 수정 불가)
class Point:
    x: float  # ← x 좌표 필드 (float 타입)
    y: float  # ← y 좌표 필드 (float 타입)

    def distance_to_origin(self) -> float:  # ← 원점(0,0)까지의 거리 계산 메서드
        return (self.x ** 2 + self.y ** 2) ** 0.5  # ← 피타고라스 정리: √(x² + y²)

# --- decorator ---
def log_call(func):  # ← 데코레이터 함수: func를 감싸서 호출 정보를 로깅
    @wraps(func)  # ← wraps로 원본 func의 __name__, __doc__ 등 메타데이터 보존
    def wrapper(*args, **kwargs):  # ← 가변 인자(*args)와 키워드 인자(**kwargs) 받기
        print(f"[LOG] calling {func.__name__} with args={args}")  # ← 호출 직전 로그 출력
        result = func(*args, **kwargs)  # ← 원본 함수 실제 실행
        print(f"[LOG] {func.__name__} returned {result}")  # ← 반환값 로그 출력
        return result  # ← 원본 함수의 결과를 그대로 반환
    return wrapper  # ← 감싼 함수(wrapper) 반환 → 데코레이터 완성

@log_call  # ← add 함수에 log_call 데코레이터 적용
def add(a: int, b: int) -> int:  # ← 두 정수를 더하는 함수
    return a + b  # ← 덧셈 결과 반환

# --- comprehension ---
squares = [x ** 2 for x in range(1, 6)]  # ← 리스트 컴프리헨션: 1²~5² → [1,4,9,16,25]
evens = {x for x in range(10) if x % 2 == 0}  # ← 셋 컴프리헨션: 0~9 중 짝수만 모으기
square_map = {x: x ** 2 for x in range(1, 6)}  # ← 딕셔너리 컴프리헨션: {1:1, 2:4, 3:9, ...}

# --- with ---
@contextmanager  # ← 제너레이터를 컨텍스트 매니저로 변환하는 데코레이터
def open_file(path: str, mode: str):  # ← 파일 경로와 열기 모드를 인자로 받음
    f = open(path, mode)  # ← 파일 열기 (리소스 획득)
    try:
        yield f  # ← with 블록에 파일 객체 f를 전달 (여기서 일시 중단)
    finally:
        f.close()  # ← with 블록 종료 후 반드시 실행: 파일 닫기 (리소스 해제)

with open_file("example.txt", "w") as f:  # ← 컨텍스트 매니저로 파일 쓰기 모드 열기
    f.write("Hello, Python!")  # ← 파일에 문자열 쓰기

# --- 실행 ---
print(add(3, 5))  # ← 데코레이터된 add 함수 호출 → [LOG] 로그와 함께 결과 8 출력
print(squares)  # ← 리스트 컴프리헨션 결과 출력
print(evens)  # ← 셋 컴프리헨션 결과 출력
print(square_map)  # ← 딕셔너리 컴프리헨션 결과 출력
print(Point(3.0, 4.0).distance_to_origin())  # ← Point(3,4)의 원점 거리 → 5.0
```

## 주요 패턴
- `@dataclass`: `__init__`, `__repr__`, `__eq__` 등을 자동 생성하여 데이터 모델링을 단순화한다.
- Comprehension: `[expr for item in iterable if cond]` 형태로 loop보다 간결하고 빠르다.
- `with` + contextmanager: `@contextmanager` 또는 `__enter__`/`__exit__`로 파일, 락 등의 자원 해제를 보장한다.
- Decorator 체이닝: `@wraps(func)`로 메타데이터를 보존하며 여러 decorator를 중첩할 수 있다.
- Generic typing: `TypeVar`와 `Generic`을 조합해 타입 안전한 제네릭 컨테이너를 정의한다.

## 주의사항
- `def` 내에서 mutable 객체(예: `[]`, `{}`)를 기본 인자로 사용하면 모든 호출이 동일 객체를 공유하므로, `None`을 기본값으로 하고 함수 내부에서 초기화하거나 `field(default_factory=list)`를 사용한다.
- Decorator를 사용할 때 `@wraps`를 생략하면 원본 함수의 `__name__`, `__doc__`이 손실되어 디버깅과 문서화에 지장이 생긴다.
- Dataclass에서 `frozen=True`로 설정하지 않으면 가변 필드가 예상치 못하게 변경될 수 있다.
- Comprehension 안에서 부작용(side effect)이 있는 함수를 호출하는 것은 지양한다.
- `typing`은 런타임에 강제되지 않으므로, 정적 분석 도구(mypy, pyright)와 함께 사용해야 효과적이다.
- `@contextmanager` 내에서 `yield` 이후의 코드는 `with` 블록이 정상 종료된 후에만 실행되며, 블록 내에서 예외가 발생해도 `finally`는 실행된다.
- `@dataclass` 필드 기본값에 `[]`나 `{}`를 직접 쓰면 모든 인스턴스가 동일 객체를 공유하므로, 반드시 `field(default_factory=list)`를 사용한다.
