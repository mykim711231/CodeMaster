# Concurrency

## Official Documentation
- [Async Execution / Task Execution](https://docs.spring.io/spring-boot/docs/current/reference/htmlsingle/#io.executors)

## 핵심 개념
> Spring Boot는 `@EnableAsync`와 `@Async`로 메서드를 별도 스레드에서 비동기 실행할 수 있습니다. `TaskExecutor`를 통해 스레드 풀을 제어하고, `CompletableFuture`로 비동기 작업 결과를 조합 및 처리합니다.  
> **동기 vs 비동기**: 동기 처리는 요청이 끝날 때까지 스레드가 블로킹되지만, 비동기 처리는 작업을 백그라운드에 맡기고 즉시 반환하므로 서버 자원을 효율적으로 사용할 수 있습니다. `CompletableFuture`는 비동기 파이프라인을 함수형 스타일(`thenApply`, `thenCompose`, `thenCombine`)로 구성할 수 있는 핵심 도구입니다.  
> **주의**: 비동기 코드는 디버깅이 어렵고 스레드 컨텍스트(인증 정보 등)가 전파되지 않으므로, 필요한 곳에만 선택적으로 적용해야 합니다.

## 학습 목표
- `@EnableAsync`와 `@Async`를 이용한 비동기 메서드 작성
- `TaskExecutor`를 커스터마이징하여 스레드 풀 설정
- `CompletableFuture`로 비동기 결과 처리 및 체이닝

## 예제 코드
```java
// ConcurrencyApplication.java — 비동기 실행을 활성화하는 메인 클래스
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;
import org.springframework.scheduling.annotation.EnableAsync;
import org.springframework.scheduling.concurrent.ThreadPoolTaskExecutor;

import java.util.concurrent.Executor;

@SpringBootApplication
@EnableAsync                                    // ← @Async 메서드를 비동기로 실행하도록 활성화해요
public class ConcurrencyApplication {

    public static void main(String[] args) {
        SpringApplication.run(ConcurrencyApplication.class, args);
    }

    @Bean(name = "taskExecutor")                // ← 커스텀 스레드 풀을 빈으로 등록해요
    public Executor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(2);            // ← 기본 스레드 수 (항상 유지)
        executor.setMaxPoolSize(5);             // ← 최대 스레드 수 (큐가 꽉 차면 확장)
        executor.setQueueCapacity(100);         // ← 대기 큐 크기 (초과 시 RejectedExecutionException)
        executor.setThreadNamePrefix("async-"); // ← 로그에서 식별하기 쉽게 스레드 이름 접두사 설정
        executor.initialize();                  // ← 모든 설정을 마친 후 명시적 초기화
        return executor;
    }
}
```

```java
// AsyncService.java — 비동기로 실행할 비즈니스 로직이에요
import org.springframework.scheduling.annotation.Async;
import org.springframework.stereotype.Service;

import java.util.concurrent.CompletableFuture;

@Service
public class AsyncService {

    @Async("taskExecutor")                     // ← 위에서 정의한 스레드 풀을 지정해서 비동기 실행
    public CompletableFuture<String> fetchData(String source) {
        try {
            Thread.sleep(2000);                // ← 2초 걸리는 작업을 시뮬레이션 (실제로는 DB/API 호출)
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt(); // ← 인터럽트 상태를 복원해요 (모범 사례)
        }
        return CompletableFuture.completedFuture("Data from " + source); // ← 완료된 Future 반환
    }
}
```

```java
// UserController.java — 비동기 결과를 클라이언트에 응답하는 컨트롤러
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

import java.util.concurrent.CompletableFuture;

@RestController
public class UserController {

    private final AsyncService asyncService;

    public UserController(AsyncService asyncService) {   // ← 생성자 주입
        this.asyncService = asyncService;
    }

    @GetMapping("/fetch")
    public CompletableFuture<String> fetch() {            // ← CompletableFuture를 그대로 반환 (non-blocking)
        return asyncService.fetchData("DB-01")
                .thenApply(result -> "Result: " + result); // ← 결과를 받아 변환하는 후속 작업 체이닝
    }
}
```

## 주요 패턴
- `thenApply()`로 비동기 결과를 변환하고 체이닝
- `CompletableFuture.allOf()`로 여러 비동기 작업을 병렬로 실행하고 결과 취합

## 주의사항
- `@Async` 메서드는 반드시 public이어야 하며, 같은 클래스 내에서 직접 호출하면 AOP 프록시가 동작하지 않음
- 반환 타입이 `void`인 `@Async` 메서드는 예외가 호출자에게 전달되지 않으므로 로깅 필수
- `ThreadLocal` 값은 비동기 스레드로 전파되지 않음 (별도 설정 필요)
- 입문자 실수: `@EnableAsync`를 깜빡하고 붙이지 않으면 `@Async` 메서드가 일반 동기 메서드로 실행됨
- 입문자 실수: 스레드 풀의 core/max/queue 설정 없이 기본값으로 운영하면 트래픽 급증 시 `OutOfMemoryError` 발생 가능
