# Java Core

## Official Documentation
- [The Java Tutorials](https://docs.oracle.com/javase/tutorial/)

## 핵심 개념
> Java는 플랫폼 독립적인 객체지향 프로그래밍 언어로, JVM 위에서 동작하며 강타입(type-safe) 언어입니다. 기본 문법, 자료형, 제어 흐름을 익히는 것이 Spring Boot 학습의 기초가 됩니다.  
> **컴파일 과정**: `.java` 소스 파일을 `javac`가 바이트코드(`.class`)로 변환하고, JVM이 이를 해석해 실행합니다. 한 번 작성하면(`Write Once, Run Anywhere`) 어떤 OS에서도 동작하는 것이 Java의 핵심 가치입니다.  
> **객체지향의 4대 원칙**인 캡슐화, 상속, 다형성, 추상화를 이해하면 Spring의 DI/IoC 패턴을 자연스럽게 받아들일 수 있습니다.

## 학습 목표
- 기본 자료형(primitive)과 참조형(reference type)의 차이 이해
- 클래스, 상속, 인터페이스를 활용한 객체지향 프로그래밍 작성
- javac/java 도구를 이용한 컴파일 및 실행

## 예제 코드
```java
// Main.java — 프로그램 진입점이에요
public class Main {
    public static void main(String[] args) {  // ← JVM이 가장 먼저 호출하는 메서드
        Calculator calc = new Calculator();   // ← 객체를 생성해서 사용해요
        System.out.println("10 + 5 = " + calc.add(10, 5));
        System.out.println("10 - 5 = " + calc.subtract(10, 5));
    }
}

interface Operation {                         // ← "이런 기능이 있어야 해"라는 계약서 역할
    int operate(int a, int b);                // ← 구현체는 반드시 이 메서드를 채워야 해요
}

class Calculator implements Operation {       // ← Operation 계약을 실제로 구현하는 클래스
    public int add(int a, int b) {            // ← 인터페이스에 없는 자체 메서드도 추가 가능
        return a + b;
    }

    public int subtract(int a, int b) {
        return a - b;
    }

    @Override                                 // ← 인터페이스의 메서드를 재정의했다는 표시
    public int operate(int a, int b) {
        return add(a, b);
    }
}
```

## 주요 패턴
- 캡슐화(Encapsulation): 필드를 private으로 선언하고 public getter/setter 제공
- 불변 객체(Immutable Object): 모든 필드를 final로 선언하고 생성자로만 초기화

## 주의사항
- `==`은 참조 비교, `equals()`는 값 비교임을 혼동하지 말 것
- null 참조로 메서드 호출 시 NullPointerException 발생에 주의할 것
- 입문자 실수: `public static void main(String args[])` — `String[] args`가 올바른 형태이며, `args`가 없으면 실행되지 않음
- 입문자 실수: 예외(Exception)를 무조건 무시(`catch (Exception e) {}`)하면 원인 불명의 런타임 오류 발생
