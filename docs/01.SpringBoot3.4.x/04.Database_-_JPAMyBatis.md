# Database - JPA/MyBatis

## Official Documentation
- [SQL Data Access (JDBC, JPA, MyBatis)](https://docs.spring.io/spring-boot/docs/current/reference/htmlsingle/#data.sql)

## 핵심 개념
> Spring Boot는 Spring Data JPA와 MyBatis를 통해 데이터베이스 접근을 추상화합니다. JPA는 `@Entity` 클래스와 `Repository` 인터페이스만으로 CRUD를 자동 생성하며, MyBatis는 XML 또는 어노테이션 기반의 SQL 매핑을 제공합니다.  
> **JPA vs MyBatis 선택 기준**: JPA는 객체 중심 설계와 단순 CRUD에 적합하고, 복잡한 통계/리포트성 쿼리는 MyBatis가 유리합니다. 두 기술을 동시에 사용할 수도 있으며, 실무에서는 JPA로 기본 CRUD를 처리하고 복잡한 조회는 MyBatis나 QueryDSL로 보완하는 패턴이 일반적입니다.  
> **영속성 컨텍스트**: JPA는 1차 캐시 역할을 하는 영속성 컨텍스트로 엔터티를 관리하며, `flush()` 시점에 변경 감지(Dirty Checking)로 UPDATE 쿼리를 자동 생성합니다.

## 학습 목표
- `@Entity`, `@Id`, `@GeneratedValue`로 JPA 엔터티 정의
- `JpaRepository` 인터페이스를 상속하여 CRUD 메서드 자동 생성
- MyBatis `@Mapper`/`@Select`로 직접 SQL 매핑

## 예제 코드
```java
// JPA 방식 — 객체를 DB 테이블과 자동 매핑해요
// User.java
import jakarta.persistence.*;

@Entity                        // ← "이 클래스는 DB 테이블이에요"라고 JPA에 알려줘요
@Table(name = "users")         // ← 실제 테이블 이름을 지정 (생략 시 클래스명 = 테이블명)
public class User {
    @Id                        // ← 기본 키(Primary Key) 컬럼이에요
    @GeneratedValue(strategy = GenerationType.IDENTITY) // ← AUTO_INCREMENT 전략 사용
    private Long id;

    @Column(nullable = false)  // ← NOT NULL 제약조건을 추가해요
    private String name;

    @Column(unique = true)     // ← UNIQUE 제약조건 (중복 이메일 방지)
    private String email;

    protected User() {}        // ← JPA가 내부적으로 사용하는 기본 생성자 (필수!)

    public User(String name, String email) {  // ← 우리가 직접 사용하는 생성자
        this.name = name;
        this.email = email;
    }

    public Long getId() { return id; }
    public String getName() { return name; }
    public String getEmail() { return email; }
}
```

```java
// UserRepository.java — JPA가 구현체를 자동 생성해줘요
import org.springframework.data.jpa.repository.JpaRepository;

public interface UserRepository extends JpaRepository<User, Long> { // ← <엔터티, PK타입>
    User findByEmail(String email);  // ← 메서드 이름만으로 SQL이 자동 생성돼요 (쿼리 메서드)
}
```

```java
// MyBatis 방식 — SQL을 직접 작성해요 (복잡한 쿼리에 유리)
// UserMapper.java
import org.apache.ibatis.annotations.*;

@Mapper                                    // ← 이 인터페이스가 MyBatis 매퍼임을 선언
public interface UserMapper {

    @Select("SELECT * FROM users WHERE id = #{id}")  // ← #{id}는 PreparedStatement 파라미터
    User findById(@Param("id") Long id);              // ← @Param으로 매개변수 이름 명시

    @Insert("INSERT INTO users(name, email) VALUES(#{name}, #{email})")
    @Options(useGeneratedKeys = true, keyProperty = "id") // ← 자동 생성된 키를 User.id에 채워줘요
    void insert(User user);
}
```

```yaml
# application.yml — 데이터베이스 연결 설정이에요
spring:
  datasource:
    url: jdbc:h2:mem:testdb            # ← H2 인메모리 DB 사용 (테스트/학습용)
    driver-class-name: org.h2.Driver   # ← JDBC 드라이버 클래스
  jpa:
    hibernate:
      ddl-auto: update                 # ← 엔터티 변경을 테이블에 자동 반영 (운영에서는 validate 권장)
```

## 주요 패턴
- Repository 패턴: `JpaRepository`를 상속한 인터페이스로 데이터 접근 계층 구현
- DTO 분리: 엔터티는 DB 매핑 전용으로 유지하고, 요청/응답용 DTO를 별도 정의

## 주의사항
- JPA 엔터티에는 기본 생성자(protected 이상)가 반드시 필요
- `@Transactional` 없이 Lazy Loading 시 LazyInitializationException 발생 가능
- MyBatis 매퍼 XML 경로와 `mybatis.mapper-locations` 설정 불일치 주의
- 입문자 실수: `ddl-auto: update`로 개발하다가 `create`/`create-drop`을 운영에 적용하면 모든 데이터가 삭제됨
- 입문자 실수: JPA `findById()`의 반환값이 `Optional<T>`임을 모르고 `.get()`을 바로 호출하면 NoSuchElementException 발생
